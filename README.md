# Flowlingo

AI-powered daily English acquisition through real-life scenarios.

> [中文文档 →](./README_zh.md)

## What is Flowlingo?

Flowlingo is an AI-powered daily English absorption tool. Users spend 10 minutes a day immersed in a real-life English scenario dialogue — reading sentence by sentence, collecting useful expressions, and doing lightweight practice. All content is generated by AI based on the user's level and interests, fresh every day.

It is **not** a course platform, a quiz bank, or an AI chat buddy. It is a structured, completable, daily English absorption loop that's different every time.

## Core Loop

```
Pick scenario → AI generates dialogue → Read sentence by sentence → Collect expressions → Practice → Review → Done
```

One loop per day. 10–15 minutes. Done and move on.

## Architecture: Agent + Skills

Flowlingo is built on an **Agent + Skills** architecture:

- **Agent** (orchestration layer): Manages the daily loop flow, tracks expression familiarity, dispatches Skills, persists data, and renders the UI. The Agent is the product itself — not portable.
- **Skills** (capability units): Independent, structured LLM prompt templates. Portable to ChatGPT, Claude, or any LLM provider.

| Skill | Input | Output |
|-------|-------|--------|
| Generate Scenario | difficulty, domain, scenario, existingExpressions | dialogue + expressions + practicePrompt |
| Evaluate Practice | practiceType, userAnswer, referenceAnswer | correct + feedback |
| Recommend Scenario | domains, recentScenarios, difficulty | scenario title + domain |

Skill prompt templates live in `src/skills/` and can be used and tested independently.

## Data Models

- **UserSettings** — difficulty (1–4), domains[], dailyMinutes (5/10/15)
- **Scenario** — AI-generated scenario with sentences and practicePrompt
- **Sentence** — English text + Chinese translation + learnable expressions
- **Expression** — Reusable English expression with familiarity tracking (new → learning → mastered)
- **PracticePrompt** — Practice exercise (comprehension / fill-in / retell)
- **DailyRecord** — Daily completion record + streak counter

All data is persisted to localStorage via Zustand.

/replay        → Expression flashcard review

Bottom navigation: **Today** · **Expressions** · **Settings**

Loop step pages (scenario → practice → replay → complete) are not in the nav bar — they are entered linearly from the Today page.

## Known Shortcomings

See the **Shortcomings** section in [README_zh.md](./README_zh.md#已知短板) for a detailed analysis, or the summary below:

1. **No component library** — All UI is inline in page files; heavy className duplication
2. **Duplicated constants** — Difficulty/domain/time options defined twice (setup + settings)
3. **No loading states** — No skeletons or spinners for async transitions
4. **No tests** — No unit or integration tests
5. **No route middleware** — Route guards are ad-hoc `useEffect` redirects in each page
6. **Accessibility gaps** — Only 1 `aria-` attribute in the entire app; no focus management
7. **Font not loaded** — Inter font referenced in CSS but never imported via `next/font`
8. **No page-level metadata** — Only root layout has `<title>`; subpages have no specific titles
9. **No animations** — No page transitions, no card flip animation, no micro-interactions
10. **PWA incomplete** — Icons exist but no `manifest.json`, no service worker
11. **No i18n** — App UI is hardcoded English; no localization infrastructure
12. **No dark mode** — Light theme only

```bash

pnpm install

# Start development server
pnpm dev

# Production build
pnpm build
pnpm start
```

Open [http://localhost:3000](http://localhost:3000) in a mobile-width browser window for the intended experience.

## AI Integration Plan

Currently using `src/services/mock-ai.ts` for simulated data. Production integration will:

1. Proxy AI calls through Next.js Route Handlers (`/api/generate-scenario`, etc.)
2. Store API keys as server-side environment variables — never exposed to the client
3. Keep AI invisible to users — no API key input, no provider selection
